Part 2: Hailstones¶

Exercise 3: Hailstone rules¶

A sequence of hailstone numbers is generated by repeatedly applying the following two rules.

  1. If the number is even, then divide it by two.

  2. If the number is odd, then multiply it by three and add 1.

  • n %% 2 gives the remainder when n is divided by 2. Experiment by replacing n in this expression with at least three different numbers.

  • Write an if-else statement that implements these rules on a numerical variable n.

Exercise 5: Hailstone sequence¶

If you load the dplyr package, you can use piping with your custom functions too. For a function with a single input this looks like INPUT %>% FUNCTION().

A hailstone sequence is built by picking a seed number, applying the hailstone function to it and then applying the hailstone function to the resulting output. Each subsequent number is produced by feeding the previous number into the hailstone function.

  • Use hailstone() and piping to produce the first seven numbers in the hailstone sequence seeded by 3.

  • Verify that your code outputs the following seven numbers: 10, 5, 16, 8, 4, 2, 1
